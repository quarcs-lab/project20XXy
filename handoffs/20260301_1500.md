## Decisions Made

| Decision | Choice | Rationale |
| --- | --- | --- |
| Which skills get `disable-model-invocation` | render, execute, init, sync-tex | These run build pipelines or modify many files — should only run when user explicitly requests |
| Audit skills auto-invocable | Yes (bib-check, data-audit, freeze-check, check-env) | Read-only, no side effects — Claude can helpfully suggest them when contextually relevant |
| `allowed-tools` per skill | Tailored per category | Read-only skills get `Bash, Read, Glob, Grep`; creation skills add `Write, Edit`; cite/literature-note add `WebSearch, WebFetch` |
| README skill organization | 6 categories with sub-tables | More scannable than one flat table; categories match the frontmatter behavior (side-effect vs auto-invocable) |
| Skill file links in README | Direct links to `SKILL.md` | Lets users read the full skill definition from the README |
