## Decisions Made

- **`{{< include >}}` for tables, `{{< embed >}}` for figures:** Quarto's embed shortcode cannot extract `display_data` markdown outputs from notebook cells. Tables are exported as `.md` files and included directly. This is documented in CLAUDE.md and all relevant skills.
- **Two-pass render:** `scripts/render.sh` runs `quarto render` twice because includes are resolved before notebooks execute. First pass generates table files, second pass includes them.
- **Manual regression tables:** `pf.etable(type="md")`, `etable(markdown=TRUE)`, and `esttab md` all produce output that doesn't render as proper HTML tables. Manual pipe-delimited markdown construction is the reliable approach.
- **6×4 inch figure standard:** Consistent across all languages (Python `figsize=(6,4)`, R `width=6, height=4`, Stata `width(1800)`).
- **No `#| label: tbl-*` on table cells:** Table labels in notebook cells conflict with the `{{< include >}}` approach. Labels are only used for figure cells.
- **Inline seeds:** Removed config.py/config.R; each notebook sets `random.seed(42)` / `set.seed(42)` / `set seed 42` directly.
